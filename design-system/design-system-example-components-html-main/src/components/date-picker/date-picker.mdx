## 仕様

### バリエーション

| 属性 | 値 | 説明 |
| --- | --- | --- |
| `data-type` | `consolidated`, `separated` | 日付ピッカーの表示タイプ |
| `data-size`（controls要素） | `sm`, `md`, `lg` | コントロールのサイズ |

### 機能仕様

#### 2タイプの日付ピッカー

1. **Consolidated（統合型）**: 年、月、日の3つの入力欄がひとつのまとまった操作領域として表示されます。キーボードの左右キーによるフィールド間のナビゲーションをサポートしています。
2. **Separated（分割型）**: 年、月、日の各フィールドが視覚的に分割されて表示されます。キーボードの左右キーによるフィールド間のナビゲーションは行えません。

#### date-picker.js の機能

- カスタム要素 `<dads-date-picker>` を定義
- Consolidatedタイプでキーボードナビゲーション（←→キーによるフィールド間移動）を実装
- カレンダーアイコンのクリックでポップアップを開閉
- カレンダーで日付を選択すると各入力欄に値を反映
- `Esc`キーによる閉じる操作、`Tab`キーによるフォーカストラップをサポート
- バリデーション機構は含まない

#### calendar.js の機能

- カスタム要素 `<dads-calendar>` を定義
- `min-date` / `max-date` 属性で日付範囲を設定可能
- 日付選択時に `date-selected` カスタムイベントを発火
- 「前月／次月」「今日」「削除」ボタンをサポート
- カレンダーUIにおける標準的なキーボードナビゲーションをサポート

## 使い方

### インストール

#### 必要なファイル

- **共通（どちらのタイプでも必要）**
  - `date-picker.css`
  - `date-picker.js`
- **カレンダー機能を使う場合に追加で必要**
  - `button.css`
  - `calendar.css`
  - `calendar.js`
  - `select.css`

スクリプトを読み込む際は必ず、`<script type="module">`を使用しES Moduleとして読み込んでください。

### ラベルのマークアップ

日付ピッカーは複数の入力フィールドから成ります。日付ピッカーにラベルを付与する際は、`<fieldset>`要素で囲んでグループ化し、グループのラベルを`<legend>`要素で提供します。

スクリーンリーダーは、グループに含まれるフィールドを読み上げる際、グループ名もあわせて読み上げます。これにより、スクリーンリーダーユーザーは入力対象について素早く理解できるようになります。

また、サポートテキストおよびエラーテキストは`aria-describedby`属性を使用して`<fieldset>`要素と紐付けを行います。サポートテキストおよびエラーテキストの`id`属性はページ内で一意にする必要があります。

```html
<fieldset class="dads-form-control-label" data-size="md" aria-describedby="support-text">
  <legend class="dads-form-control-label__label">
    生年月日
    <span class="dads-form-control-label__requirement" data-required="true">※必須</span>
  </legend>
  <p id="support-text" class="dads-form-control-label__support-text">
    西暦で記入してください。<br>
    例）2021年09月01日
  </p>
  <div>
    <dads-date-picker class="dads-date-picker" data-type="consolidated">
      ...
    </dads-date-picker>
  </div>
</fieldset>
```

#### 参考情報

- [H71: fieldset 要素及び legend 要素を使用して、フォームコントロールのグループに関する説明を提供する [WCAG 2.1]](https://waic.jp/translations/WCAG21/Techniques/html/H71)

### エラー状態の実装

エラーをユーザーに表示する場合は、各入力フィールドに以下の属性を設定します。

- `aria-invalid="true"`: 入力が不正であることをスクリーンリーダーに伝える
- `aria-describedby="<エラーテキストのid>"`: エラーメッセージ要素のIDを参照

```html
<input class="dads-date-picker__input" type="text" aria-invalid="true" aria-describedby="error-text" data-js-year-input>
```

Consolidatedタイプの場合は、外側のコンテナに`data-error`属性を追加してスタイルを変更します。

```html
<div class="dads-date-picker__inputs" data-error>
  ...
</div>
```

### 読み取り専用状態

読み取り専用にする場合は、`readonly`属性を追加し、コンテナに`data-readonly`属性を設定します。

```html
<div class="dads-date-picker__inputs" data-readonly>
  <label class="dads-date-picker__year">
    <span class="dads-date-picker__label">年</span>
    <input class="dads-date-picker__input" type="text" value="2021" readonly data-js-year-input>
  </label>
  ...
</div>
```

### 無効状態

無効にする場合は、各入力フィールドに`disabled`属性を追加し、コンテナに`data-disabled`属性を設定します。

```html
<div class="dads-date-picker__inputs" data-disabled>
  <label class="dads-date-picker__year">
    <span class="dads-date-picker__label">年</span>
    <input class="dads-date-picker__input" type="text" disabled data-js-year-input>
  </label>
  <label class="dads-date-picker__month">
    <span class="dads-date-picker__label">月</span>
    <input class="dads-date-picker__input" type="text" disabled data-js-month-input>
  </label>
  <label class="dads-date-picker__day">
    <span class="dads-date-picker__label">日</span>
    <input class="dads-date-picker__input" type="text" disabled data-js-day-input>
  </label>
</div>
```

### バリデーション

`date-picker.js`には入力値のバリデーション機構を含みません。すべての入力チェック（数値であること、妥当な範囲であることなど）は、開発者側でスクリプトをカスタマイズするか、サーバーサイドで行う必要があります。
